# Roteiro de Vídeo - Apresentação de até 3 minutos

## 1. Abertura
"Olá, este é o CardioIA na Fase 5, um protótipo de assistente cardiológico inteligente e conversacional desenvolvido para fins acadêmicos na FIAP."

## 2. Problema e objetivo
"A proposta desta fase foi construir um chatbot educacional capaz de realizar uma triagem inicial simulada, orientar o usuário sobre sintomas cardiovasculares e identificar cenários de urgência, sem realizar diagnóstico médico."

## 3. Arquitetura
"A solução foi estruturada com frontend simples em HTML, CSS e JavaScript puro, backend em Flask e integração com IBM Watson Assistant por meio da API v2. Para garantir executabilidade mesmo sem credenciais, incluímos um fallback local para demonstração offline."

## 4. Demonstração do fluxo
"Na interface, o usuário informa sintomas como dor no peito, palpitação ou falta de ar. O backend valida a entrada, consulta o serviço do assistente e aplica regras de segurança. Se houver sinais de alerta, como dor intensa no peito com irradiação ou desmaio, o sistema exibe orientação imediata para buscar atendimento médico."

## 5. Governança e ética
"Todo o projeto foi desenvolvido com foco em responsabilidade. O assistente informa claramente que não faz diagnóstico, não prescreve medicamentos e não substitui avaliação profissional. Também há preocupação com governança, ética e princípios alinhados à LGPD."

## 6. Encerramento
"Como resultado, entregamos um protótipo funcional, organizado para GitHub e pronto para evoluções futuras em experiências conversacionais na área da saúde cardiovascular."
